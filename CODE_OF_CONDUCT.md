# Code of Conduct

StemStudio welcomes contributors who want to improve a FOSS local music tool without turning it into a hosted product.

## Expected Conduct

- Keep discussion focused on the project, the user workflow, and the code.
- Treat other contributors with basic respect, even when you disagree.
- Give actionable feedback. Point to the file, behavior, command, or screenshot that explains the issue.
- Respect privacy. Do not post private audio, generated stems, cookies, tokens, local paths, or library databases.
- Assume maintainers may reject work that adds product scope the project does not support.

## Unacceptable Conduct

- Harassment, insults, threats, or personal attacks.
- Publishing someone else's private data, media, credentials, or local filesystem details.
- Repeatedly pushing unrelated product direction after a maintainer has closed the topic.
- Submitting intentionally harmful code, dependency changes, or instructions.

## Enforcement

Maintainers may edit, close, or delete comments and issues that violate this code. They may block contributors who keep disrupting the project.

Report conduct problems by opening a GitHub issue if the report can be public. If the report includes private details, contact the maintainer through the public contact link in the maintainer's GitHub profile.
